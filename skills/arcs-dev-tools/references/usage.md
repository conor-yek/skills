# arcs-dev-tools 使用示例与触发语句

本文件用于**技能发现**与**用户触发**示例，按需加载，不占用主 `SKILL.md` 的篇幅。

## 典型触发语句（用户可能会这样说）

- “帮我拉取 ARCS SDK 仓库并初始化 submodule”
- “我的环境没装好，缺 cmake / riscv64-unknown-elf-gcc，帮我装工具链”
- “编译这个 sample（指定路径/示例名）并给我固件产物路径”
- “把这个 `arcs.bin` 烧录到板子上（串口是 /dev/ttyACM0）”
- “烧录后跑起来，读一下串口日志，判断是否通过”
- “串口设备号变了/被占用/权限不足，帮我排查”

## 最小流水线示例

### 示例 1：跑通一个现有 sample

1. 定位 sample 目录（由代码代理自行完成）
2. 用 `SKILL.md` 的**操作 3**编译
3. 用 **操作 4**烧录
4. 用 **操作 5**读日志并返回给代码代理判定

### 示例 2：反复迭代 UI 文案验证

1. 代码代理修改 UI 文案
2. 工具链代理编译 → 烧录 → 读日志/观察现象
3. 不符合预期则回到步骤 1
